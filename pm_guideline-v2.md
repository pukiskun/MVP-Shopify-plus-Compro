# Project Guideline: Project Manager (PM)

## Project Context
Company Profile & E-Commerce Website (Security-First Approach). 
Core Features: Detailed product catalog, Automated Shipping Calculation (J&T API), and Payment Gateway (Xendit). All visual assets use placeholders.

## Main Focus
Manage project scope, timeline, ensure compliance with data security regulations, and bridge communication between the Fullstack Developer and QA.

## Responsibilities & Workflow
1. **Credential & API Management:**
   * Responsible for registering sandbox accounts for **Xendit** and **J&T API**.
   * Ensure API Keys (especially Secret Keys) are securely distributed to the Developer. Sending keys via open group chats is strictly prohibited; use encrypted password managers (e.g., 1Password/Bitwarden).

2. **Sprint Definition & Feature Prioritization:**
   * Divide the development phase into sprints:
     * **Sprint 1:** Company Profile, Product Catalog (placeholders), and basic shopping cart.
     * **Sprint 2:** J&T shipping rate integration, Xendit UI/Invoice integration, and Security Testing.

3. **Documentation & Security Regulations:**
   * Draft a Privacy Policy regarding the use of customer data (such as address and phone number) that will be shared with third parties (J&T and Xendit).

4. **Release Readiness Validation:**
   * Act as the final decision-maker for production release based on QA reports. 
   * **Critical Rule:** Release must be postponed if any High/Critical level vulnerabilities remain.
